## 项目背景(供 AI 理解上下文)

四川麻将识别系统,两阶段架构。本 Phase 只做**检测器**:2 类(`tile_face` / `tile_back`),不做 29 类分类(那是 Phase 3)。

**最终验收目标(在 Phase 0 冻结测试集上)**:
- 较短边 >20px 的牌:召回 ≥99.5%,精确率 ≥98%
- 较短边 10~20px 的牌:召回 ≥95%
- 空桌面/无关画面误检率极低(后续会定义负样本测试)

**技术路线**:YOLO11n/s + P2 小目标检测头,输入 960~1280;先纯合成数据训练 → 评测找差距 → 伪标注真实帧 + 人工校正 → 混合微调 → 滚动迭代 2~3 轮。

**硬件**:单张消费级 GPU(24GB 显存级别),所有训练配置以此为约束。

### 评测纪律(必须遵守,写进 README 显眼位置)

- **冻结测试集 `test_set_v1` 只用于里程碑评测**(每轮迭代结束跑一次),绝不参与训练、不做早停依据、不做超参搜索依据
- 日常训练监控用独立的 **dev 验证集**(本 Phase 任务 2 中构建,来自真实视频帧但与测试集来源视频不重叠)
- 所有进入训练/dev 集的帧,必须通过 `get_leaked_videos()` 排除测试集来源视频(复用 Phase 1 的实现)

## 任务 1:项目脚手架 + 迭代操作手册

**做什么**:建目录、写 `paths.yaml`、写 `docs/iteration_playbook.md`。

**`iteration_playbook.md` 内容**:一页纸说清一轮完整迭代的人机分工流程:
1. 训练(AI 脚本)→ 2. dev 集评测看曲线(AI 脚本,我看数字)→ 3. 伪标注新一批真实帧(AI 脚本)→ 4. 我在 CVAT 校正(人工,附校正操作要点:删误检/补漏检/调框/改 face-back,目标速度 30~60 秒/张)→ 5. 合并数据重训(AI 脚本)→ 6. 里程碑跑冻结测试集 + badcase 分析(AI 脚本,我看报告定下一步)
并写明:什么情况下该补合成数据(某博主域整体差 → Phase 1 加该域风格背景/模板)、什么情况该补真实标注(特定 badcase 模式)、什么情况该停止迭代(连续两轮冻结测试集提升 <0.3%)。

**验收**:我读完能独立执行一轮迭代,无歧义。

## 任务 2:dev 验证集构建 `scripts/build_dev_set.py`

**做什么**:从博主视频构建 300~400 张的 dev 验证集,作为日常训练监控标尺(冻结测试集的"平替",可以脏一点但来源分布要相近)。

**功能需求**:
1. 排除泄漏视频后,按 Phase 0 任务 3 相同的"博主配额 + phash 去重"逻辑选帧(复用代码)
2. 标注方式:旧 YOLO11 模型伪标注(置信度 ≥0.5)直接作为"银标准",**不要求人工精校**——dev 集允许 ~5% 噪声,只用于看相对趋势;在 README 明确此局限
3. 可选 `--cvat-export` 输出 CVAT 预标注包,若我后续愿意花 2 小时精校则升级为金标准
4. 输出 COCO 格式 + 来源分布报告;每张图带 `source` 字段(博主)

**验收标准**:dev 集与 test_set_v1 零视频重叠(脚本自动校验并打印证明);博主分布大致均衡。

## 任务 3:模型结构与训练管线 `configs/yolo11_p2.yaml` + `scripts/train.py`

**做什么**:基于 Ultralytics YOLO11 实现加 P2 检测头的 2 类检测器训练。

**功能需求**:
1. **模型配置**:在 YOLO11n 与 YOLO11s 的官方 yaml 基础上增加 P2 输出层(stride 4),保留 P3/P4,**移除 P5 大目标头**(本任务没有占半个画面的牌,省算力给小目标);配置文件加注释说明每处修改
2. **训练入口 train.py**:
   - 读 `train_base.yaml`:imgsz(默认 960,支持 1280)、batch(自动按显存探测最大值)、epochs、优化器、增广开关(注意:Phase 1 合成数据已含外观增广,训练侧 mosaic/mixup 保留但 HSV 增广强度减半,避免双重增广失真;此逻辑写注释)
   - 支持 `--data` 传入任意 COCO 数据集目录(自动转 YOLO 格式缓存,转换脚本含在内)
   - 每次运行自动创建 `runs/exp_{日期}_{名称}/`,内含:完整配置快照、训练曲线(tensorboard 或 csv+png)、最优/最终权重
   - 训练中按 epoch 在 dev 集上评测 mAP@0.5 与小目标桶召回(写自定义 callback),曲线入日志
3. **两段式训练预设**:`--preset synth_pretrain`(纯合成,从 COCO 预训练权重起,~80 epochs)与 `--preset real_finetune`(从上一次权重起,低学习率,~30 epochs)

**验收标准**:在合成集 det_synth_v1 上完整训练能跑通且 GPU 利用率 >85%;dev 集指标曲线正常上升;P2 配置加载无报错且模型 summary 显示 stride 4 输出层存在;同配置两次训练(固定 seed)指标差异 <0.5%。

## 任务 4:一键评测 `scripts/run_eval.py`

**做什么**:把任意权重在指定测试集(dev 或冻结测试集)上的评测打包成一条命令,底层调 Phase 0 的 eval_detection.py。

**功能需求**:
1. `run_eval.py --weights xxx.pt --dataset dev|frozen|<path>`:推理(含 TTA 开关)→ 生成 COCO 预测 json → 调 Phase 0 评测 → 报告归档到对应 exp 目录
2. 推理参数(conf/iou/imgsz)与训练配置解耦,可在命令行覆盖,评测报告中记录所用参数
3. 对 `frozen` 数据集加一道确认提示("冻结测试集评测属于里程碑操作,确认?y/N"),并在 `runs/frozen_eval_log.csv` 追加一条记录(日期/权重/关键指标),强制留痕防滥用

**验收标准**:一条命令出完整 HTML 报告;frozen 评测留痕机制生效。

## 任务 5:伪标注与主动学习选帧 `scripts/pseudo_label.py`(本 Phase 人工降本核心)

**做什么**:为下一轮人工校正,自动挑"最值得标"的真实帧并生成 CVAT 预标注。

**功能需求**:
1. 排除泄漏视频与已入库(real_det_v* 及 dev)的帧;对候选视频抽帧(复用 Phase 0 extract_frames 的质量过滤)
2. 用当前最优权重推理,conf 阈值放低到 0.2(宁多勿漏,人工删框比画框快)
3. **主动学习打分**,每帧综合分 = 加权和:
   - 不确定性:框置信度落在 0.25~0.6 区间的数量占比(越多越值得标)
   - 小目标含量:<25px 检出框占比
   - 域稀缺性:该博主在已有 real 训练数据中的占比越低,加分越高
   - 多样性惩罚:与已选帧 phash 相近的降权
4. 按综合分取 Top-N(`--num 500`,默认每轮 500 张),打包成 CVAT task zip(每 50 张一个 task,预标注随包导入),输出导入操作说明
5. 输出选帧报告:分数分布、博主分布、预计校正工时估算(按 45 秒/张)

**验收标准**:选出的帧人眼看确实"难"(模糊/远牌/遮挡/新桌布居多);CVAT 导入后预标注框正确显示;与已有数据零重复。

**人工配合**:我在 CVAT 校正,每轮 500 张约 6 小时;校正完导出 COCO。

## 任务 6:真实数据入库与混合集构建 `scripts/merge_real_data.py`

**做什么**:把我校正导出的 COCO 合并入库,并组装训练用混合集。

**功能需求**:
1. 入库:校正后 COCO 先过 Phase 0 validate_coco.py,通过后存为 `real_det_v{N}`(N 自增),生成 MANIFEST(帧数/框数/博主分布/对应校正批次)
2. 混合集组装 `mixed_v{N}`:参数化配比 `--real-ratio`(默认按 真实:合成 = 1:3 的有效采样比);实现方式用**符号链接 + 重复采样清单**(真实数据通过在清单中重复出现来提升采样权重),不物理复制图像
3. 自动校验混合集总量、配比、无泄漏

**验收标准**:混合集能被 train.py 直接消费;配比与配置一致(脚本打印实际统计);磁盘不产生图像副本。

## 任务 7:badcase 归因分析 `scripts/analyze_badcase.py`

**做什么**:每轮里程碑评测后,自动归类错误,告诉我"下一步该补什么"。

**功能需求**:
1. 输入:评测产生的预测 json + GT;提取全部 FN(漏检)与 FP(误检)
2. 自动归因标签(一个错误可多标签):小目标(<20px)/ 重遮挡(与邻框 IoU>0.3)/ 模糊(crop Laplacian 低)/ 画面边缘 / 横放旋转牌 / face-back 混淆 / 按博主聚类
3. 输出 HTML 报告:各归因桶的数量与占比、每桶 20 张可视化样例(GT 绿框/预测红框)、**与上一轮报告的桶间对比**(哪类错误在收敛、哪类没动)
4. 末尾自动生成"建议行动"段落:基于规则映射(如"博主 X 的 FN 占比 >30% → 建议 Phase 1 补该博主背景素材并增加伪标注配额")

**验收标准**:对一份真实评测结果运行,归因标签抽查 20 例准确率 ≥90%;两轮对比视图正确。

## 任务 8:模型导出 `scripts/export_model.py`

**做什么**:把通过验收的最优权重导出为下游(Phase 3 crop 来源、Phase 4 集成、Phase 5 部署)可用的格式。

**功能需求**:
1. 导出 ONNX(opset 12+,动态 batch 关闭、固定输入尺寸,部署友好);可选 FP16
2. **精度对齐检查**:同一批 100 张图,.pt 与 ONNX 推理结果逐框比对(IoU>0.99 且 conf 差 <0.01 视为一致),一致率 ≥99% 才算导出成功
3. 调 Phase 0 benchmark_fps.py 出延迟报告,随导出物归档
4. 产出物目录含:onnx 文件、输入输出规格说明(尺寸/归一化/输出格式解码方式)、对齐与速度报告——下游 Phase 拿这个目录即可集成,不需要回头翻训练代码

**验收标准**:对齐检查通过;规格说明文档完整到"不看训练代码也能写出推理端解码"。

## 全局技术约束

- Python 3.10+;核心依赖:ultralytics, torch, opencv-python, pycocotools, imagehash, pyyaml, tqdm, onnx, onnxruntime, tensorboard
- 单卡 24GB 显存为上限设计;所有长任务支持断点续跑
- 实验可追溯:任何一个 runs/exp_* 目录必须自包含(配置快照 + 数据集版本号 + git commit hash),删掉其他目录也能说清这次实验是什么
- 复用 Phase 0/1 已有代码(评测、校验、去重、抽帧),路径经 paths.yaml 配置,不复制粘贴实现
- pytest:数据转换(COCO↔YOLO)、混合集采样清单、主动学习打分函数必须有单测
