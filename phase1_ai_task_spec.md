# Phase 1 需求规格:麻将识别项目 — 合成训练数据生产管线

> 本文档交给 AI 编码助手(opencode / codex / Claude Code)执行。
> 请严格按任务顺序实现,每个任务有明确的输入、输出和验收标准。
> 全部代码用 Python 3.10+,依赖写入 requirements.txt,每个脚本支持 `--help`。
> 本 Phase 依赖 Phase 0 产物:`mahjong-eval/configs/classes.yaml`、`data/raw_videos/`(博主视频)、旧 YOLO11 模型。

## 项目背景(供 AI 理解上下文)

我在做一个第一人称视角的实时**四川麻将**识别系统,两阶段架构:
- 检测器:YOLO11(2 类:`tile_face` / `tile_back`)
- 分类器:29 类(w1~w9 / t1~t9 / b1~b9 / back / unknown),不含字牌花牌

Phase 1 的目标:建立一条**全自动 2D 合成数据管线**,产出:
1. 检测训练集:3~5 万张合成场景图,COCO 格式,2 类标注
2. 分类训练集:50 万张带标签的牌面 crop,按类分文件夹

核心要求:**远小牌样本(较短边 10~30px)占比 ≥40%**,这是旧模型的主要短板。

素材来源策略:**牌面/牌背模板主要从已下载的博主视频中自动提取**(我人工精选),桌面背景同样从视频帧提取 + 网图补充。我手边可能有实体麻将可拍照补充模板,作为可选输入支持。

### 与 Phase 0 的衔接

- 类别定义直接读取 `mahjong-eval/configs/classes.yaml`,**不要重新定义**,路径可配置
- 提取素材所用视频 = Phase 0 抓取的博主视频,但**必须排除已进入冻结测试集 `test_set_v1` 的那些帧所在的视频片段**(读取测试集 COCO 的 image 文件名反查来源视频,整段视频排除,防止数据泄漏)。实现一个公共函数 `get_leaked_videos()` 供本 Phase 所有脚本调用

## 任务 1:项目脚手架 + 模板精选指引文档

**做什么**:创建目录结构、`paths.yaml`(含 mahjong-eval 各路径配置项),并写 `docs/template_curation.md`。

**`template_curation.md` 内容要求**:
- 我在 `templates_candidate/` 各类文件夹里精选模板的标准:正面朝向(透视畸变小)、较短边 ≥60px、无遮挡、无高光糊版、色彩正常;每类挑 **5~15 张**,刻意保留不同牌款式/磨损程度/轻微光照差异的多样性
- 操作方式:文件管理器大图标模式,把选中的复制到 `templates/{类名}/`,不合格的不用删
- `back` 模板的挑选要点(牌背花纹种类尽量多,不同博主的牌背都要有)
- 可选实拍补充流程:若有实体麻将,每类正面拍 3~5 张(正光/侧光/微俯角),放入 `templates/photo/{类名}/`,管线会自动抠图合并
- 干扰物素材说明:从视频帧里截手部/骰子/茶杯等小图存入 `distractors/`,10~20 张即可,提升合成真实度

**验收**:目录完整;文档我照做无歧义。

## 任务 2:模板候选自动提取 `scripts/extract_templates.py`

**做什么**:扫描博主视频,自动收集高质量牌面 crop 作为模板候选,按类预分到 29 个文件夹,把我的人工从"拍照抠图"降为"挑图"。

**功能需求**:
1. 调用 `get_leaked_videos()` 排除测试集来源视频
2. 对每个视频按 2 fps 采样,用旧 YOLO11 模型(`--model`)检测;只保留:置信度 ≥0.7、较短边 ≥60px、框完整在画面内的牌
3. crop 时外扩 8% 边距;按旧模型预测类别放入 `templates_candidate/{类名}/`(预测为字牌的丢弃)
4. **质量过滤**:Laplacian 清晰度阈值、过曝/欠曝过滤(同 Phase 0 任务 2 标准,复用代码思路)
5. **同款去重**:phash 去重(同一博主同一牌反复出现),每个"phash 组"只留清晰度最高的 1 张;每类候选上限 200 张(`--per-class-cap`),按清晰度排序保留
6. **自动抠图**:对每张候选用 rembg 或 OpenCV GrabCut 生成带 alpha 的 PNG(牌是规则矩形,可先用最小外接四边形检测 + 透视矫正到标准正视图,再抠背景,矫正后的模板质量更高——优先实现这条路线)
7. 输出 `extract_templates_report.json`:每类候选数、来源博主分布;某类候选 <10 张要在控制台醒目警告

**验收标准**:对 2~3 个真实博主视频运行,候选文件夹里大部分图类别正确、抠图边缘干净、已透视矫正为正视图;报告完整。

**人工配合**:跑完后我按任务 1 文档精选,预计 1~2 小时。

## 任务 3:背景与干扰物提取 `scripts/extract_backgrounds.py`

**做什么**:从博主视频帧中收集"可用作合成画布"的桌面背景。

**功能需求**:
1. 排除泄漏视频;按 0.2 fps 采样
2. 用旧模型检测,选"牌很少"的帧(检出 ≤3 张牌),对检出的牌区域用 OpenCV inpaint 抹除,得到干净桌面背景
3. 输出到 `backgrounds/`,按博主分子目录;phash 去重;目标每个博主 ≥20 张、总计 ≥150 张
4. 支持 `--import-dir` 把我从网上找的桌面纹理图批量归一化(resize 到统一最长边 1920、统一 jpg)并入库
5. 输出统计报告

**验收标准**:背景图人眼看无明显残留牌影;各博主都有背景入库。

## 任务 4:合成参数配置 `configs/synth_config.yaml` 设计

**做什么**:先于合成引擎,定义一份完整的、带注释的参数配置文件,所有合成行为由它驱动(便于后续迭代 synth_v2/v3 只改配置)。

**必须包含的参数组**(每项给默认值和注释):
1. **场景布局**:每图牌数范围(8~40);布局模式及采样权重——`hand_row`(手牌 13~14 张紧密横排,近距大牌)/ `river_grid`(牌河 6×N 网格,中距,允许横牌)/ `wall`(牌墙双层,远距,大量 tile_back)/ `meld`(碰杠 3~4 张组)/ `scatter`(少量随机散牌);一张图可组合多种布局
2. **尺寸分布**:目标牌尺寸直方图——10~20px / 20~30px / 30~60px / >60px 的占比(默认 20/25/30/25,保证小牌 ≥45% 留余量);引擎必须按此分布反推贴牌缩放
3. **几何**:透视抖动范围(模拟第一人称俯角 15°~60°)、旋转(±8° 常规 + 横牌 90°)、同布局内牌间重叠率(0~30%)
4. **遮挡**:手部/干扰物贴图遮挡概率与遮挡比例上限(70%,与标注规范一致);被遮挡牌仍输出完整可见部分的框
5. **外观**:亮度/对比度/白平衡抖动、高光斑(模拟反光)、阴影投射(贴牌底部软阴影,去"浮空感"的关键)、运动模糊(方向+长度)、高斯噪声、JPEG 压缩质量范围(50~90,模拟视频压缩)
6. **画布**:输出分辨率(1280×720 与 1920×1080 混合)、背景采样策略
7. **类别平衡**:27 个牌面类出现频次均衡约束(最大/最小类频次比 ≤1.5)

**验收标准**:配置文件自带 schema 校验函数(非法值给出明确报错);README 里有"改哪个参数影响什么"的速查表。

## 任务 5:检测集合成引擎 `scripts/synth_detection.py`(本 Phase 核心交付)

**做什么**:读模板库 + 背景库 + synth_config.yaml,批量生成带 COCO 标注的合成场景图。

**功能需求**:
1. 按任务 4 的布局模式组装场景:先选背景 → 按布局放置牌(带透视一致性:同一图内所有牌共享一个全局视角参数,近大远小符合同一灭点,**不要每张牌独立随机透视,否则画面不真实**)→ 叠干扰物/遮挡 → 全图统一外观增广
2. 牌背(`tile_back`)合成:牌墙与对手立牌场景大量使用 back 模板
3. 标注生成:每张牌输出 bbox(被遮挡的按可见区域)+ 检测类别 + **同时在 annotation 的自定义字段记录其分类标签**(后续 crop 复用);严格 COCO 格式,与 Phase 0 评测脚本兼容
4. 性能:多进程并行,生成速度 ≥10 张/秒(1080p,普通 8 核机器);支持 `--num 50000 --seed 42` 可复现
5. 尺寸分布闭环:生成结束输出实际尺寸直方图,与配置目标对比,偏差 >5% 报警
6. 每张图的元数据(用了哪些模板/背景/布局)写入 COCO image 自定义字段,便于排查

**验收标准**:生成 200 张样图,人眼检查:无浮空感、布局像真实牌局、远牌确实小且糊得自然、标注框严丝合缝(用 Phase 0 的 validate_coco.py 校验通过);5 万张生成可在 2 小时内完成;相同 seed 两次生成结果一致。

## 任务 6:分类 crop 集合成 `scripts/synth_classification.py`

**做什么**:生成 50 万张分类训练 crop,27 类 + back(unknown 类不合成,由真实数据贡献)。

**功能需求**:
1. 两个来源混合:(a)从任务 5 的合成场景图按标注框 crop(带场景上下文边缘,占 60%);(b)模板直接增广生成(占 40%,速度快)
2. **关键退化增广**:随机先缩到 12~40px 再放大回 96×96(模拟远牌检测后 crop 放大的真实输入,这是分类器能认远牌的核心);透视、部分裁切(模拟检测框不准)、遮挡贴片、反光高光
3. 类别严格均衡;输出按类分文件夹 + 一个 labels.csv
4. 同样支持 seed 复现、多进程

**验收标准**:每类抽 50 张人眼检查类别无误、退化程度覆盖"清晰到勉强可认"的完整谱系;50 万张 1 小时内生成完毕。

## 任务 7:合成质量预览 `scripts/preview_synth.py`

**做什么**:生成单文件 HTML 预览页,辅助我快速质检每个版本的合成数据。

**功能需求**:
1. 检测集:随机 50 张大图(可开关显示标注框)+ 按布局模式分组各 10 张 + 尺寸分布直方图
2. 分类集:每类一行 × 10 张随机 crop 的缩略图墙
3. **真假对照区**:左边合成图、右边从真实视频帧随机抽的图,各 20 张并排,供我主观判断风格差距
4. 无外网 CDN 依赖

**验收标准**:打开即用,我 5 分钟内能完成一轮质检。

## 任务 8:数据集打包与版本管理 `scripts/package_dataset.py`

**做什么**:把合成产物固化为带版本号的数据集目录。

**功能需求**:
1. 输出 `det_synth_v{N}/`(images/ + annotations.json)与 `cls_synth_v{N}/`;版本号自动递增
2. 每个版本生成 MANIFEST.md:生成时间、所用 synth_config.yaml 的完整快照、模板库/背景库的文件清单 hash、图像数量与尺寸分布、随机 seed——保证任何版本可完整复现
3. 校验:打包前自动跑 Phase 0 的 validate_coco.py,不通过则中止

**验收标准**:打包两个版本后,MANIFEST 足以让我说清两版差异;COCO 校验通过。

## 全局技术约束

- Python 3.10+;核心依赖:opencv-python, numpy, pillow, albumentations, rembg(或 GrabCut 方案则不需要), pyyaml, tqdm, ultralytics(仅推理用), imagehash, jinja2
- 复用优先:质量过滤、phash 去重等逻辑 Phase 0 已有实现思路,保持参数与行为一致
- 所有脚本 argparse + `--help`;长任务有进度条与断点续跑(已生成的不重复生成)
- 合成引擎核心几何/标注计算必须有 pytest 单测(构造已知布局验算 bbox)
- 不引入 Blender/3D 渲染(本期 2D 方案;若 Phase 2 评测显示 sim2real 差距过大,再立 3D 升级项)
