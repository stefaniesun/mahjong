# X-AnyLabeling 标注操作指南(麻将识别项目)

> 配套 Phase 0 标注规范使用。本文解决三个问题:工具怎么配、流程怎么走、框怎么画才对。
> 注:快捷键以软件"帮助→快捷键"页面为准,不同版本可能微调,本文写的是常用默认值。

---

## 一、安装与项目配置(一次性,15 分钟)

1. **下载**:GitHub 搜 `CVHub520/X-AnyLabeling`,Releases 页下载 Windows 版(带 GPU 的版本用于模型辅助标注更快,CPU 版也够用);免安装,解压即用。界面支持中文(设置里切换)
2. **类别文件**:启动后通过"上传→上传标签文件"导入 `configs/xanylabel_classes.txt`(29 类,顺序与 classes.yaml 一致);这样画框后弹出的类别列表是固定下拉,避免手输错拼写
3. **必开的三个设置**:
   - **自动保存**(文件→自动保存):每张图的标注实时写同名 JSON,防止白干
   - **保留上一帧类别**:连续标同类时少点一次
   - 文件列表面板打开(View 菜单),方便看进度和跳图
4. **打开数据**:文件→打开目录,指向 `frames_selected/`;如果目录里已有 `make_prelabel.py` 生成的预标注 JSON,框会自动显示出来——你的工作就是改,不是画

## 二、标注工作流(两遍法)

**核心原则:一次只做一种判断。** 框的位置判断和类别判断混着做,速度慢且都容易错。

### 第一遍:只管框,不管类(每张图 20~40 秒)

逐张过,只做四个动作:
- **删误检**:点选错误框(或框列表里点)→ Delete。预标注 conf 放得低,误检偏多是正常的,删比画快
- **补漏检**:按 `W` 进入画矩形模式,两个对角点拉框;漏检集中在远处小牌和重遮挡牌,**养成习惯:每张图最后扫一眼画面上半部(远处牌墙/对家区域),那里最容易漏**
- **调歪框**:点选框后拖动四角/四边;预标注的框普遍可用,只调明显不贴边的
- 此遍类别先不纠结,模型预测的类别错了也先放着

### 第二遍:只管类(每张图 10~20 秒)

利用左侧标注列表逐框过类别,或用"按类别过滤显示"功能一类一类检查:
- 重点过:t4/t5、t6/t9、t2/t3、万字组、b2/b3——这些是模型和人都容易错的
- 改类:选中框 → 回车(或双击列表项)→ 下拉选正确类别
- 远小牌放大确认:`Ctrl+滚轮` 缩放,放大到牌面占屏幕 1/10 以上再判断;放大后仍 1 秒内认不出 → 改 `unknown`,不要硬猜

### 导航与节奏

- `D` 下一张 / `A` 上一张;改完即走,自动保存兜底
- **25 分钟一休**:标注错误率在疲劳后陡增,宁可多休息;每天首尾各抽 5 张昨天的图复查,自查漂移
- 全部完成后跑 `export_to_coco.py` 转换,再跑 `validate_coco.py`,看 `validation_preview/` 里的可视化抽查图做最终质检

## 三、框怎么画才对(本项目的画框规则)

### 基本规则:框住"整个可见牌体",贴边但不贴死

- 框边界 = 牌的可见外轮廓,**包含牌的侧壁**(麻将牌是有厚度的立体,立着的牌能看到正面+顶部侧沿,躺着的牌能看到牌面+朝向你的侧壁)——整个牌体都框进去,因为检测器学的是"一张牌"的完整外观,侧壁的颜色和质感也是有效特征
- **不包含**:桌面上的阴影、强反光在桌布上的光斑、相邻牌的任何部分
- 容差 ±2 像素即可,不必像素级强迫症;但系统性偏松(每个框都多框 5~10px 桌面)或偏紧(切掉牌边)是大忌,会教坏模型的定位精度

### 各种具体情形

| 情形 | 画法 |
|---|---|
| 正常平放/立放的牌 | 贴可见轮廓的轴对齐矩形 |
| 斜放/旋转的牌 | 仍画轴对齐矩形,框住整个倾斜牌体(四角会带进少量背景,正常,不要为了"紧"切掉牌角) |
| 叠牌/相邻紧贴 | **每张牌单独一框**,允许框互相重叠;不要一个大框框一排 |
| 部分遮挡(手/其他牌) | 只框**可见部分**的外接矩形;遮挡 >70% 仍要框(人眼能确认是牌就框),类别认不出给 unknown |
| 画面边缘截断的半张牌 | 照框,框到图像边界为止 |
| 极小远牌(<10px) | 放大后按可见特征判断:能确认具体万/条/筒就标对应类别;能确认是牌背就标 `back`;看不清具体类别时标 `unknown` |
| 立着的牌墙(双层) | 上层牌逐张框 back;下层只露一条边的,露出部分 ≥牌宽 30% 才框,否则跳过(标注规范的可辨认底线) |
| 运动模糊严重的帧 | 牌的轮廓糊成一片无法定位边界的,该帧本不该入选;个别糊牌框大致轮廓+unknown |

### 小目标的画法技巧(本项目质量关键)

远处小牌是这个项目的核心战场,框的质量直接决定 P2 头能学多好:
1. **必须放大后画**:`Ctrl+滚轮` 放大到该牌至少 50px 显示大小再拉框;在 100% 缩放下徒手框 12px 的牌,误差能到 30%,等于喂噪声
2. 小牌密集区(牌河远端、对家牌墙)建议用"放大→框完一片区域→平移→下一片"的扫格子式走法,不要全图视野下东一个西一个地点
3. 框完缩回全图视野再扫一眼,确认没有漏掉孤立的小牌

### 类别判定速查(分类相关)

- 能稳定认出具体万/条/筒牌面,就标对应的 27 个具体类别(`w1~w9`、`t1~t9`、`b1~b9`)
- `back`:只见清晰牌背纹理,能明确不是牌面
- 分类 `unknown` 的三种情况:① 只见侧棱立面;② 可见牌面 <30% 或糊到 1 秒认不出;③ 字牌/花牌(东南西北中发白、春夏秋冬梅兰竹菊)
- **拿不准就 unknown**:错标一张具体类的危害 > 少标一张训练样本

## 四、模型辅助标注(可选进阶)

当前仓库主流程以 `scripts/make_prelabel.py` 生成预标注为准。除此之外,X-AnyLabeling 也支持在软件内直接跑模型:
1. 左侧 AI 按钮 → 模型列表选"加载自定义模型",写一个模型配置 yaml(指向你最新导出的 ONNX、类别表、置信度阈值)——Phase 2 每出一版新模型,替换 ONNX 路径即可,预标注质量会随训练进步自动变好
2. 内置 SAM/SAM2 可框选辅助:对轮廓清晰但预标注漏掉的牌,点一下牌面自动出框,再微调
3. 工具→批量运行模型:这是软件内重生成预标注的替代做法,与脚本预标注二选一即可;**不要对已人工校正过的目录重跑**,会覆盖你的成果——先备份

## 五、质检与常见翻车点

- **每批交叉复查 20%**:隔天用"随机抽样"的眼光重看(或换个人看),重点查:远牌漏框、unknown 与具体类的边界一致性、t6/t9 方向
- 常见翻车:
  1. 忘了导入类别文件,徒手输 label 造成 `t4`/`T4`/`4t` 混杂 → 转 COCO 时报未知类别;开工前务必导类别文件
  2. 对已校正目录重跑批量模型,覆盖人工成果 → 校正完的批次目录立刻只读/归档
  3. 一个大框框一排手牌 → validate_coco 的异常宽高比检查会抓到,但最好画的时候就别犯
  4. 框里带进大片阴影 → 影响检测器对牌边界的学习,贴轮廓
  5. 中途换判定标准(前 200 张侧棱算 back、后 300 张算 unknown)→ 开工前把本文第三节读两遍,中途有拿不准的情形,记到一个 questions.md 里统一裁决并回溯修改,不要各标各的
